# 开发规范：Setup + TypeScript 模式

## 一、核心规范

**新项目/新功能必须使用 `setup` 语法糖 + TypeScript 开发**

## 二、基础模板

```vue
<template>
    <div class="component-name">
        <!-- 模板内容 -->
    </div>
</template>

<script setup lang="ts">
import { ref, reactive, computed, onMounted } from "vue";

// 1. 定义 Props
interface Props {
    title: string;
    count?: number;
}

const props = withDefaults(defineProps<Props>(), {
    count: 0,
});

// 2. 定义 Emits
const emit = defineEmits<{
    (e: "update", value: number): void;
}>();

// 3. 响应式数据
const state = reactive({
    dataList: [] as any[],
});

const message = ref("");

// 4. 计算属性
const doubleCount = computed(() => props.count * 2);

// 5. 方法
const handleClick = () => {
    emit("update", props.count + 1);
};

// 6. 生命周期
onMounted(() => {
    console.log("组件挂载");
});
</script>

<style lang="scss" scoped>
/* 样式 */
</style>
```

## 四、类型定义规范

```typescript
// 接口定义
interface User {
    id: string;
    name: string;
    age?: number;
}
```

## 五、最佳实践

### 5.1 组件命名

- 文件名：`kebab-case`（如 `chart-form.vue`）
- 组件引用：`PascalCase`（如 `ChartForm`）

### 5.2 类型安全

- 所有函数参数必须声明类型
- 响应式数据尽量提供类型推断
- 使用 `type` 或 `interface` 定义复杂类型

### 5.4 避免的写法

```typescript
// ❌ 不推荐：any 类型
const data = ref<any>([]);

// ✅ 推荐：明确类型
const data = ref<Item[]>([]);

// ❌ 不推荐：魔法数字
if (status === 1) {
}

// ✅ 推荐：枚举常量
if (status === Status.ACTIVE) {
}
```

---

**执行标准**：所有新开发的 `.vue` 文件必须遵循此规范，确保代码一致性和类型安全性。
